# Agent Collaboration & Traceability

## Collaboration Goal

Enable multiple agents to work concurrently while preserving consistency, quality, and auditability.

## Conventions

1. **Module ownership**
   - Intake, Retrieval, Reasoning, Recommendation, Memory, Feedback each have clear owners.
2. **Single source docs**
   - Product, architecture, knowledge-base, and roadmap docs are canonical references.
3. **Schema-first updates**
   - Update shared input/output schemas before implementation changes.
4. **Minimal-change discipline**
   - Keep PR scope narrow and tied to a defined objective.

## Traceability Practices

For each recommendation generated by the system, persist:
- Recommendation ID
- Source evidence IDs
- Applied framework IDs
- Assumptions used
- Confidence level
- Reviewer decision (approved/edited/rejected)
- Outcome feedback link

For each case-study extraction drafted by `case_study_extractor`, persist:
- Source material IDs and locators
- Evidence excerpts mapped to schema fields
- Prompt version and extractor agent name
- Uncertainty notes and manual-review requirement
- Final curator decision before dataset inclusion

For each code/documentation change, include:
- Linked objective/issue
- Modules affected
- Validation performed
- Open follow-up items

## Human-in-the-Loop Review Rules

Mandatory review when:
- Legal, tax, compliance, or financing implications exist
- Confidence is low or evidence is weak
- Advice affects irreversible decisions

Reviewer checklist:
- Evidence quality
- Assumption validity
- Risk disclosure completeness
- Action clarity and sequencing

## Handoff Template (Agent to Agent)

- Objective
- Scope boundaries
- Inputs consumed
- Outputs produced
- Decisions made and rationale
- Remaining risks / open questions
- Next recommended actions

## Case-Study Curation Workflow

1. **Research agent or curator gathers source material**
   - Capture `source_id`, title, URL, source type, publication year, and locator details.
2. **`case_study_extractor` produces a draft**
   - Output must match the shared case-study schema and include `evidence`, `source_materials`, and `uncertainty_notes`.
3. **Human review validates the draft**
   - Remove unsupported claims.
   - Confirm the outcome framing is balanced for both successful and failed businesses.
   - Keep `manual_review_required=true` until the draft is approved.
4. **Curated record is stored in `data/case_studies/`**
   - Retain extraction metadata for auditability even after approval.
