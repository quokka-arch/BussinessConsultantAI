# Implementation Roadmap

## MVP Definition

Primary job-to-be-done:
> Help early-stage founders evaluate a business idea and choose the next experiments.

Target niche:
- Early-stage SaaS and online businesses

MVP inputs:
- Idea, target customer, monetization concept
- Stage, geography, constraints, team context

MVP outputs:
- Idea scorecard
- Top comparable businesses
- Top risks and assumptions
- Validation experiments
- 30-day execution plan

## Phase Plan

### Phase 1 (Weeks 1–2): Foundation

- Finalize niche and report schema
- Prepare intake form fields
- Curate initial case-study dataset
- Define baseline evaluation rubric

### Phase 2 (Weeks 3–4): MVP Build

- Implement intake -> retrieval -> report workflow
- Generate standardized consulting reports
- Add source traces and confidence labels
- Test with pilot founders

### Phase 3 (Weeks 5–6): Quality Loop

- Add human reviewer workflow
- Capture usefulness feedback and outcomes
- Improve weak sections (generic/wrong advice)
- Add follow-up chat for experiment tracking

### Phase 4: Operator Assistant

- Weekly check-ins
- KPI trend review and prioritization support
- Expanded playbooks for pricing, GTM, hiring

### Phase 5: Strategic Advisor

- Multi-quarter memory and scenario planning
- Strategy memo generation
- Early risk and opportunity alerts

## Evaluation, Risks, and Guardrails

### Evaluation Criteria (score 1–5)

- Relevance
- Factual grounding
- Strategic usefulness
- Actionability
- Clarity
- Risk awareness
- Non-genericness

### Key Risks

- Overconfident bad recommendations
- Hallucinated examples or unsupported claims
- Generic advice with low execution value
- Legal/financial overreach

### Guardrails

- Require citations where possible
- Distinguish fact vs inference
- Always include confidence level
- Trigger human review for high-stakes guidance
- Add explicit legal/tax/financial professional-review disclaimer
